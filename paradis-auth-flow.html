System Architecture

Ymir — Auth Server

apps/ymir/ • :3001

OAuth 2.1 Authorization Server with Better Auth. Handles all identity operations, social login, and token management.

  • OAuth 2.1 + OIDC
  • Google / Apple / LinkedIn
  • Credentials login
  • Signup + Email verify
  • Forgot password
  • Token SSO
  • Encrypted JWT storage

Next.js — Client App

apps/nextjs/ • :3000

Stateless client using JWE-encrypted session cookies. Talks to ymir via OAuth 2.1 and OIDC discovery.

  • Stateless JWE sessions
  • proxy.ts route guard
  • OAuth Proxy (Vercel)
  • SSO entry point
  • Token refresh link
  • AsyncLocalStorage

Three Independent Token Stores
🔒

Better Auth Session

__Secure-better-auth.session_token

JWE-encrypted cookie containing user identity and session metadata. Managed by Better Auth framework.

🔑

DLAI Auth Cookie

__Secure-dlai_auth

JSON cookie with DLAI JWT token, OAuth access/refresh tokens, user ID and hash. Used for API calls.

🗄

Database Record

user.dlaiJwtToken (encrypted)

JWE-encrypted DLAI JWT stored on user record. Single source of truth. Decrypted by customSession plugin.

Authentication Flows
OAuth 2.1 Social Login (PKCE)
User clicks "Sign in with Google/Apple/LinkedIn" — full PKCE-protected flow through ymir back to Next.js.
1 Browser Navigate to protected route
User visits /my/* (any protected path)
2 Next.js proxy.ts intercepts — no session found
Calls auth.api.signInWithOAuth2(), generates PKCE code_verifier + state cookie
proxy.ts → redirect to ymir /api/auth/oauth2/authorize?response_type=code&code_challenge=...
3 Ymir OAuth authorize endpoint
Validates client_id, redirect_uri, PKCE params. Redirects to social provider.
4 Google / Apple / LinkedIn User authenticates with provider
User enters credentials at provider. Provider redirects back to ymir callback.
5 Ymir handleOAuthAccount() hook fires
Exchanges provider token with DLAI upstream API (/user/login-by-google, etc.) to get DLAI JWT. Encrypts JWT with JWE using AUTH_SECRET. Stores on user record.
OAuth token → DLAI API → dlaiJwtToken → JWE encrypt → user.dlaiJwtToken
6 Ymir Issues authorization code
Creates Better Auth session in DB. Redirects to Next.js callback with auth code.
7 Next.js Callback: exchange code for tokens
Exchanges auth code at ymir /oauth2/token (with PKCE code_verifier). Gets access_token, refresh_token, id_token.
8 Next.js Fetch user info + write cookies
getUserInfo() calls ymir /oauth2/userinfo, stages DLAI claims via AsyncLocalStorage. After hook writes __Secure-dlai_auth cookie with tokens.
userinfo → AsyncLocalStorage.run() → after hook → Set-Cookie: __Secure-dlai_auth={dlaiJwtToken, accessToken, refreshToken, ...}
9 Browser Authenticated! Two cookies set
Browser now has __Secure-better-auth.session_token (JWE) + __Secure-dlai_auth (JSON). Redirected to original protected page.
Email / Password (Credentials)
Direct credential login via ymir's custom credentialsPlugin. Bypasses OAuth flow.
1 Browser User submits email + password
Login form on Next.js sends to ymir
2 Ymir POST /api/auth/sign-in/credentials
credentialsPlugin() calls DLAI API /user/login-by-password with email/password.
3 DLAI API Validates credentials, returns JWT
Returns dlaiJwtToken + user profile data if credentials valid.
4 Ymir Token staging pattern
Encrypts DLAI JWT → stores on user record before creating session. Clears sessionData cache (maxAge: 0) to prevent stale reads.
encrypt(dlaiJwt) → user.dlaiJwtToken → clear sessionData cookie → create session
5 Browser Receives session + token, redirects
Client receives token + callbackURL, initiates OAuth flow to establish Next.js cookies, or uses direct session.
SSO — External Platform Token Auth
External DLAI platform passes JWT to Next.js, which validates and creates a session. No password or OAuth needed.
1 External Platform Redirect with JWT
Navigates to /auth/sso?token=JWT&callbackURL=/courses/chatgpt
2 Next.js Validate JWT locally
Checks JWT expiration (5-min grace period) and audience claim (aud === "WEB").
app/auth/sso/route.ts → decode JWT → check exp + aud
3 Ymir POST /api/auth/sign-in/token
tokenPlugin() validates JWT via DLAI API /user/profile. Creates/updates user in DB.
4 Next.js Set session cookies
Writes JWE session cookie + dlai_auth cookie. Redirects to callbackURL.
5 Browser Authenticated, lands on course page
Token Refresh (Transparent)
When DLAI JWT expires mid-session, the tRPC authRefreshLink intercepts the 401 and silently refreshes tokens.
1 Browser tRPC call returns UNAUTHORIZED (401)
DLAI JWT has expired. tRPC authRefreshLink catches the error.
trpc/auth-refresh-link.ts → onError: UNAUTHORIZED → trigger refresh
2 Next.js POST /api/auth/refresh-session
Calls shared refreshDlaiSession() from @dlai/auth.
3 @dlai/auth Exchange refresh token at ymir
Reads refresh_token from dlai_auth cookie. Exchanges at ymir /oauth2/token. Fetches fresh claims from /oauth2/userinfo.
refreshToken → ymir /oauth2/token → new access_token → ymir /oauth2/userinfo → fresh claims
4 Next.js Update dlai_auth cookie
Writes updated cookie with fresh tokens and claims. Returns 200.
5 Browser Retry original tRPC call
authRefreshLink retries the failed call. If refresh failed, redirects to OAuth login.
Cookie Architecture
__Secure-better-auth.session_token
Format: JWE (encrypted)
Contains: user identity, session ID
Encrypted by: SESSION_SECRET (Next.js) / AUTH_SECRET (ymir)
Flags: httpOnly, Secure, SameSite=Lax
Max-Age: 30 days (sliding)
__Secure-dlai_auth
Format: JSON (httpOnly)
Contains: dlaiUserId, dlaiJwtToken, dlaiUserHash, accessToken, refreshToken, idToken
Written by: Next.js after hook (via AsyncLocalStorage)
Flags: httpOnly, Secure, SameSite=Lax
Max-Age: 30 days
PKCE State Cookie
Format: Encrypted state
Contains: code_verifier, redirect URI
Lifetime: Transient (cleared after callback)
Set by: proxy.ts during OAuth initiation
sessionData (Better Auth cache)
Format: Cached session data
Purpose: Reduces DB lookups (7-day cache)
Gotcha: Cleared (maxAge: 0) after credentials login to prevent stale reads
Managed by: Better Auth cookieCache plugin
Special: Vercel Preview OAuth Proxy
Preview URLs (*.vercel.app) aren't registered with ymir. The oAuthProxy plugin encrypts cookies through the stable deployment.
1 Preview Detect origin mismatch
oAuthProxy plugin detects preview URL doesn't match registered redirect_uri.
2 Stable Deploy Override redirect_uri to stable URL
Encrypts state cookie + preview URL. Uses stable deployment's registered redirect_uri.
3 Ymir OAuth flow as normal
Redirects back to stable deployment (registered URI).
4 Stable Deploy Encrypt cookies into query param
Encrypts all Set-Cookie headers into ?cookies=<encrypted>. Redirects to preview URL's /api/auth/oauth-proxy-callback.
5 Preview Decrypt and set cookies on preview domain
Decrypts cookies using SESSION_SECRET (must match stable deployment). Sets them on preview domain.
Security Patterns

PKCE Protection

All OAuth flows use Proof Key for Code Exchange. code_verifier stored in transient cookie, code_challenge sent to authorize endpoint.

JWE Token Encryption

DLAI JWT tokens encrypted at rest on user record using AUTH_SECRET. Session cookies encrypted with SESSION_SECRET. Keys must differ.

Token Staging Pattern

Encrypted token written to user record BEFORE session creation. Prevents race condition where session reads stale data from cache.

Cookie Security

All auth cookies: httpOnly, Secure, SameSite=Lax, __Secure- prefix in production. 30-day sliding expiration.

Log Sanitization

Pino logger with custom serializers. JWT patterns auto-detected and replaced with [REDACTED_JWT]. Sensitive fields excluded.

OIDC Logout

RP-Initiated Logout with id_token_hint. Ymir revokes session on both sides. Clears all local cookies before redirect.
