Three-layer caching system

Cache Layers

Layer What It Caches Control
Cloudflare Static assets, HTML, APIs Cache-Control headers
Vercel Edge ISR pages, RSC, fetch results revalidate property
Next.js ISR HTML, App Router data revalidate, cache flags

How to Check Cache Status

Using cURL:
curl -I https://learn.deeplearning.ai/
Using Browser DevTools:
  • Open DevTools (F12)
  • Go to Network tab
  • Select a request
  • View Response Headers

Key HTTP Headers

Header Purpose Example Values
cf-cache-status Cloudflare cache result HIT, MISS, DYNAMIC
x-vercel-cache Vercel cache result HIT, MISS
cache-control Caching directives max-age=3600
age Time in cache (seconds) 199, 1255506

Cloudflare: Homepage

https://learn.deeplearning.ai/
cf-cache-status: DYNAMIC x-vercel-cache: HIT cache-control: public, max-age=0, must-revalidate age: 199 server: cloudflare
Interpretation:
  • Cloudflare bypasses cache (DYNAMIC)
  • Vercel serves from cache
  • Page was cached 199 seconds ago

Cloudflare: Static Asset

https://learn.deeplearning.ai/learnext/_next/static/chunks/e7791de1-b84b808d99d4e7c5.js
cf-cache-status: HIT x-vercel-cache: HIT cache-control: public, max-age=31536000, immutable age: 1255506 server: cloudflare
Interpretation:
  • Both layers cache the asset
  • Cached for 1 year (31536000s)

Vercel: Homepage

Cache Status Values

Status Meaning
HIT Served from cache (fast)
MISS Not in cache, fetched from origin
DYNAMIC Cloudflare bypasses cache
EXPIRED Cache expired, needs revalidation

Cache Flow: HIT

1. User requests static asset
↓
2. Cloudflare cache: HIT
↓
3. Response served instantly

Fastest path: Cloudflare edge

Cache Flow: MISS

1. User requests new page
↓
2. Cloudflare: MISS
↓
3. Vercel Edge: MISS
↓
4. Next.js generates page

Slowest path: Full generation

Cache Flow: DYNAMIC

1. User requests homepage
↓
2. Cloudflare: DYNAMIC (bypass)
↓
3. Vercel Edge: HIT
↓
4. Response from Vercel cache

Common for ISR pages

Best Practices

Static Assets:
max-age=31536000, immutable
ISR Pages:
Set appropriate revalidate intervals
Dynamic Content:
max-age=0, must-revalidate
Monitoring:
Check age and cache status headers

Key Takeaways

1. Three cache layers work together
2. Check cf-cache-status and x-vercel-cache
3. Static assets: long TTL + immutable
4. ISR pages: max-age=0, must-revalidate
5. Use age to check cache freshness
